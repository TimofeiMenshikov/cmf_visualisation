# CMF visualization

Инструменты для эксперимента по подбору цветового соответствия, визуализации гамута CIE 1931, записи измеренных интенсивностей и построения CMF/метрик по файлу `data.txt`.

Актуальная точка входа для работы с гамутом - `experiment.py`. Отдельный скрипт `plot_gamut.py` удален и больше не описывается в руководстве.

## Что есть в папке

| Файл | Назначение |
| --- | --- |
| `experiment.py` | Основное интерактивное окно эксперимента. Управляет гамутом, спектрами, AO-устройством, сохранением строк в `data.txt`. |
| `gamut.py` | Логика графиков, точек гамута, слайдеров, пересчета яркостей и частот/мощностей для каналов. |
| `plot_cmf.py` | Построение CMF по `data.txt`, сохранение `normalized_cmf.txt`, расчет метрик `Delta E`, SAM, NMAE и графиков проверки. |
| `XYZ_gamut_creator.py` | Инструмент для подбора/сохранения преобразований и геометрии гамута в rg/XYZ. |
| `ao_color_setter.py` | Обертки для переключения AO-каналов в статическом или попеременном режиме. |
| `constants.py` | Основные длины волн, стартовые яркости, пути к калибровке и коэффициенты нормировки. |
| `ui_constants.py` | Цвета и координаты UI-элементов. |
| `data.txt` | Экспериментальные точки. Используется для расчета CMF. |
| `normalized_cmf.txt` | Результат нормировки CMF, создается/обновляется `plot_cmf.py`. |

## Зависимости

Код ожидает Python 3 и соседнюю папку `ao-system`, потому что импортирует:

- `ao.ao_device`
- `ao.spectral_converter`
- калибровочные данные из `ao-system/ao/calibration/...`

Основные Python-библиотеки:

- `numpy`
- `scipy`
- `matplotlib`
- `colour-science`

Перед запуском убедитесь, что активировано нужное окружение и зависимости установлены.

## Быстрый запуск

Из корня проекта:

```bash
python3 cmf_visualization/experiment.py
```

По умолчанию `experiment.py` запускает:

```python
AoDeviceExperiment(read_output=True, simulate=True)
```

То есть программа работает в режиме симуляции AO-устройства. Для реального устройства нужно изменить аргумент `simulate` в `main()` на `False` и убедиться, что устройство подключено и калибровочные пути в `constants.py` корректны.

## Основной экран эксперимента

Окно состоит из нескольких зон:

- слева: диаграмма CIE 1931 с точками primaries, mono и суммарными цветами;
- справа сверху: информационная панель текущего состояния или метрик;
- справа посередине: поле `Lambda mono`, кнопки и слайдеры;
- справа снизу: график `Spectra` или CMF.

Вся интерактивная визуализация гамута теперь находится здесь, отдельного окна из `plot_gamut.py` больше нет.

### Поле `Lambda mono`

Введите длину волны монохроматического канала и нажмите Enter. Допустимый диапазон в `experiment.py`:

- минимум: `443.4`
- максимум: `743.6`

После изменения программа пересчитывает яркости, обновляет графики и частоты/мощности AO-каналов.

### Кнопки

| Кнопка | Действие |
| --- | --- |
| `Show CMF` / `Show Spectra` | Переключает правый нижний график между спектрами и CMF. |
| `show_metrics` / `show_intensity` | Переключает правую верхнюю панель между состоянием эксперимента и метриками. |
| `Save` | Добавляет текущую точку в `data.txt`, если она еще не была сохранена после последнего изменения. |
| `Reset` | Пересчитывает стартовые яркости для текущей `Lambda mono` и обновляет графики. |
| `Y+` | Умножает все яркости на текущий коэффициент `scale`. |
| `Y-` | Делит все яркости на текущий коэффициент `scale`. |

### Слайдеры

| Слайдер | Действие |
| --- | --- |
| `Y_STEP` | Шаг изменения яркости с клавиатуры. |
| `scale` | Коэффициент масштабирования для кнопок `Y+` и `Y-`. |

## Горячие клавиши

Клавиши работают, когда фокус находится в окне графика.

| Клавиша | Действие |
| --- | --- |
| `r` / `R` | Увеличить / уменьшить `Y_R`. |
| `g` / `G` | Увеличить / уменьшить `Y_G`. |
| `b` / `B` | Увеличить / уменьшить `Y_B`. |
| `i` / `I` | Увеличить / уменьшить `Y_m`. |
| `m` / `M` | Увеличить / уменьшить `Lambda mono` на 1 нм. |
| `y` | Отправить в AO каналы `red + green`. |
| `Y` | Отправить в AO каналы `blue + mono`. |
| `q` | Закрыть окно. |

Уменьшение яркости не уводит значение ниже нуля. При изменении `Lambda mono` код избегает совпадения с длинами волн primaries.

## Формат `data.txt`

Каждая строка содержит одну экспериментальную точку:

```text
lambda_m I_R I_G I_B I_m
```

Где:

- `lambda_m` - длина волны монохроматического канала;
- `I_R`, `I_G`, `I_B`, `I_m` - интенсивности для красного, зеленого, синего и монохроматического каналов.

Пример:

```text
549 36.9342 118.5804 5.8057 116.5983
```

Кнопка `Save` дописывает текущую строку в конец `data.txt`.

## Типовой рабочий процесс

1. Запустить `experiment.py`.
2. Выставить нужную `Lambda mono` через поле ввода или клавиши `m`/`M`.
3. Подобрать яркости каналов клавишами `r/R`, `g/G`, `b/B`, `i/I`.
4. При необходимости менять `Y_STEP` и общий `scale`.
5. Проверять спектры на правом графике.
6. Нажать `Save`, когда точка подобрана.
7. Повторить для других длин волн.
8. Нажать `Show CMF` или запустить `plot_cmf.py`, чтобы построить CMF по сохраненным данным.

## Расчет CMF и метрик

Построение CMF по текущему `data.txt`:

```bash
python3 cmf_visualization/plot_cmf.py
```

Скрипт:

- читает `data.txt`;
- добавляет теоретические точки primaries;
- нормирует CMF;
- сохраняет `normalized_cmf.txt`;
- строит диагностические графики;
- считает метрики цветового соответствия.

В интерфейсе `experiment.py` кнопка `show_metrics` также считает метрики и выводит их в правую верхнюю панель.

## Вспомогательная геометрия гамута

Для экспериментов с преобразованиями rg/XYZ используйте:

```bash
python3 cmf_visualization/XYZ_gamut_creator.py
```

Этот скрипт отвечает за отдельные геометрические построения и подбор матриц. Он не заменяет основной экспериментальный интерфейс и не пишет точки в `data.txt`.

## Калибровка и основные настройки

Основные параметры находятся в `constants.py`:

- `LAMBDA_RED`, `LAMBDA_GREEN`, `LAMBDA_BLUE` - длины волн primaries;
- `LAMBDA_M_START` - стартовая монохроматическая длина волны;
- `Y_STEP_START` - стартовый шаг изменения яркости;
- `Y_R_START`, `Y_G_START`, `Y_B_START`, `Y_M_START` - стартовые яркости;
- `CALIBRATION_PATH` - CSV с калибровкой амплитуда/интенсивность;
- `TABLE_SPECTRA_PATH` - папка с таблицами спектров;
- `L_R`, `L_G`, `L_B` - коэффициенты нормировки CMF.

Если меняется AO-устройство или набор калибровок, в первую очередь проверьте `CALIBRATION_PATH` и `TABLE_SPECTRA_PATH`.

## Частые проблемы

### Не находится модуль `ao`

Проверьте, что рядом с `cmf_visualization` есть папка `ao-system`, и что запуск идет из корня проекта:

```bash
python3 cmf_visualization/experiment.py
```

### Нет калибровочных файлов

Проверьте пути в `constants.py`. Сейчас они указывают на калибровку:

```text
../ao-system/ao/calibration/2026-02-18/...
```

### `Save` не добавляет новую строку

После сохранения текущей точки флаг `IS_SAVED_DATA` становится `True`. Чтобы сохранить еще одну строку, измените яркость или `Lambda mono`, затем нажмите `Save` снова.

### График CMF выглядит странно

Проверьте:

- достаточно ли точек в `data.txt`;
- нет ли дубликатов/ошибочных строк;
- не перепутаны ли столбцы `I_R`, `I_G`, `I_B`, `I_m`;
- соответствуют ли коэффициенты `L_R`, `L_G`, `L_B` текущему эксперименту.

## Безопасность данных

Перед серией экспериментов полезно сделать копию текущего `data.txt`, например `data_old.txt` или файл с датой. `Save` дописывает данные в конец файла и не очищает его автоматически.
